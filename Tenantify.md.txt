# Tenantify 🏢 — Ruby Gem Development Plan

> **Modern multi-tenancy for Rails, solving every real-world problem acts_as_tenant couldn't.**

---

## 1. Why Tenantify?

`acts_as_tenant` is outdated, unmaintained in key areas, and missing modern Rails ecosystem support. Tenantify is a full replacement built for Rails 7+, covering every pain point developers face today.

---

## 2. Problems We're Solving

### From acts_as_tenant GitHub Issues

| Issue | Problem | Our Solution |
|-------|---------|--------------|
| #356 | Sidekiq retries lose tenant context | Auto re-attach tenant on retry |
| #354 | `update_all`, `delete_all` not tenant-scoped | Scope ALL write operations |
| #353 | GoodJob loses `current_tenant` | First-class GoodJob support |
| #352 | Solid Queue not working | Built-in Solid Queue support |
| #342 | Cross-tenant associations not validated | Auto cross-tenant association validation |
| #338 | `without_tenant` broken in RSpec | Proper RSpec helpers built-in |
| #335 | ActiveJob testing broken | Test helpers for all job adapters |
| #330 | ActiveStorage blocked | Native ActiveStorage support |
| #329 | Tenant ID silently overridden | Raise error + audit log |
| #328 | Sidekiq tenancy clobbered between jobs | Unified job middleware |

### Modern World Problems (Not in acts_as_tenant)

| Problem | Our Solution |
|---------|--------------|
| No Hotwire/Turbo awareness | Tenant-scoped Turbo Streams |
| No GraphQL support | Tenant context in GraphQL queries |
| No API mode support | Header / JWT based tenant resolution |
| No subdomain + custom domain together | Both subdomain AND custom domain routing |
| No tenant switching for admins | `switch_tenant!` for superadmins |
| No per-tenant config | Each tenant has its own settings hash |
| No tenant seeding | `tenantify:seed` rake task |
| No console helpers | Rails console tenant helpers |

## 3. DSL Vision

```ruby
# Model
class Project < ApplicationRecord
  include Tenantify::Scoped
  belongs_to_tenant :organization
end

# Controller
class ApplicationController < ActionController::Base
  include Tenantify::Controller

  set_tenant_by :subdomain       # auto from subdomain
  # set_tenant_by :header        # X-Tenant-ID header (API mode)
  # set_tenant_by :jwt           # decode from JWT
  # set_tenant_by :custom_domain # custom domain mapping
end

# Jobs — all queues supported
class ReportJob < ApplicationJob
  include Tenantify::Job  # works with Sidekiq, GoodJob, Solid Queue

  def perform
    current_tenant  # always available, survives retries
  end
end

# Admin tenant switching
Tenantify.switch_to(organization) do
  # everything scoped to that org
end

# Per-tenant config
organization.tenantify_config[:max_users]      # => 50
organization.tenantify_config[:storage_limit]  # => "10GB"

# RSpec helpers
RSpec.describe Project, type: :model do
  include Tenantify::TestHelpers

  with_tenant(:organization) do
    it "scopes correctly" { ... }
  end
end
```

## 5. Module Breakdown

### 5.1 `Tenantify::Scoped` (Model Concern)
- Automatically applies `where(tenant_id: current_tenant)` on all queries
- Validates associations don't cross tenant boundaries
- Protects `update_all`, `delete_all`, `destroy_all` with tenant scope
- Raises `Tenantify::TenantMismatchError` on silent override attempts

### 5.2 `Tenantify::Controller` (Controller Concern)
- `set_tenant_by :subdomain / :header / :jwt / :custom_domain`
- Sets `current_tenant` for the duration of the request
- Supports multiple resolvers with fallback chain
- Raises `Tenantify::TenantNotFoundError` with configurable behavior (redirect / 404 / raise)

### 5.3 `Tenantify::Job` (Job Concern)
- Serializes tenant ID into job payload
- Restores tenant context on retry — **fixes acts_as_tenant #356**
- Unified support for:
  - **Sidekiq** — middleware auto-registered
  - **GoodJob** — execution hooks
  - **Solid Queue** — dispatcher hooks
  - **ActiveJob** — base adapter fallback

### 5.4 Resolvers
| Resolver | Use Case |
|----------|----------|
| `Subdomain` | `acme.yourapp.com` → `Organization.find_by(subdomain: "acme")` |
| `Header` | `X-Tenant-ID: 42` header for API clients |
| `JWT` | Decode `tenant_id` from JWT payload |
| `CustomDomain` | `acme.com` → lookup custom domain mapping table |

### 5.5 `Tenantify::Switcher` (Admin)
```ruby
Tenantify.switch_to(org) { User.all }    # block form
Tenantify.switch_to!(org)                # persistent switch (console use)
Tenantify.without_tenant { User.all }    # bypass tenant scope
```

### 5.6 `Tenantify::Auditor`
- Logs every `current_tenant=` assignment with caller info
- Raises `TenantOverrideError` when tenant is changed mid-request
- Configurable: `:log`, `:raise`, or `:ignore`

### 5.7 `Tenantify::Configuration` (Per-Tenant Config)
```ruby
Tenantify.configure do |config|
  config.tenant_model = "Organization"
  config.on_tenant_not_found = :raise   # :raise, :redirect, :null_tenant
  config.audit_overrides = :log
end

# Per-tenant settings stored as JSONB
organization.tenantify_config[:max_users] = 100
organization.tenantify_config.save!
```

### 5.8 Test Helpers
```ruby
# RSpec
include Tenantify::TestHelpers
with_tenant(org) { ... }
without_tenant { ... }

# Minitest
setup { Tenantify::TestHelpers.set_tenant(org) }
teardown { Tenantify::TestHelpers.clear_tenant }
```
